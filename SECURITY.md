# 安全策略

## 支持版本

当前仅维护最新发布版本。发现安全问题后，请先升级到最新版本并确认问题仍可复现。

## 报告漏洞

请优先通过 GitHub 仓库的“Security”页面创建私密安全报告，不要在公开 Issue、群聊或日志截图中披露 Cookie、代理凭据、压缩包密码或完整利用细节。

报告时建议提供受影响版本、运行平台、最小复现步骤、预期行为和已脱敏日志。维护者确认后会评估影响、准备修复并在适当时发布安全更新。

## 凭据泄露

如果 Cookie、代理密码或其他访问令牌曾被提交到公开仓库，应立即在对应服务中撤销或轮换；仅从 Git 历史中删除文本并不能使旧凭据失效。

管理页面默认导出脱敏配置。启用“导出敏感配置”后，生成文件会包含 ZIP 密码、Cookie 和代理信息，应仅保存在受信任设备中，并在使用后及时删除。

管理页面的一次性登录不会保存 JM 账号和密码，但凭据仍需经过浏览器与 AstrBot 服务端传输。部署在非本机环境时应启用 HTTPS，并限制管理页面只能由受信任管理员访问。登录成功后仅 Cookie 会写入插件配置；如怀疑管理端、日志或配置文件泄露，应立即轮换密码并撤销登录态。
